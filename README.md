#  Retail Sales Analysis (SQL Project)

##  Project Overview
This project presents a comprehensive retail sales analysis conducted using SQL across four relational tables: `products`, `customers`, `orders`, and `order_items`.

The objective was to extract actionable business insights from transactional retail data using advanced SQL querying, segmentation logic, and relational joins to support data-driven decision-making.

---

##  Dataset Structure

The retail database consists of:

- 100 Orders  
- 100 Customers  
- 100 Products  
- 5 Cities Represented  
- **Total Revenue: $248,226**

The schema design supports normalized relational querying and multi-table joins for advanced business intelligence extraction.

# Strategic Recommendations

1. **Set up a monthly revenue alert threshold**  
   Monitor revenue volatility and trigger alerts when performance drops below expected benchmarks.

2. **Introduce a loyalty reward program**  
   Target customers who cross-shop across multiple categories to improve retention and lifetime value.

3. **Increase regional digital promotions**  
   Focus marketing efforts on top-performing cities to maximize ROI.

4. **Upselling Strategy Training**  
   Train sales and customer service teams to propose complementary products to customers approaching the $1,000 threshold.

5. **Expand Premium Electronics Inventory**  
   Increase stock of high-performing electronics products to capture demand.

6. **Introduce Extended Warranties & Bundles**  
   Improve average order value through add-on services and product bundling.
